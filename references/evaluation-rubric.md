# 短剧剧本爆款评估体系 — 四维十六指标

> 来源：`evaluation.xlsx`（操作者提供的评分表）。evaluator 评估官在里程碑门禁使用；
> 各创作 agent 在自检时对照内容层指标。每指标 1-10 分，加权汇总后换算百分制定级。

## 维度与指标

### 市场层（权重 25%）
| 指标 | 权重 | 评估标准 |
|---|---|---|
| 受众清晰度 | 10% | 核心看剧人群是否可画像（性别、年龄、地域、付费习惯） |
| 题材窗口期 | 7% | 当前平台该题材供需比（蓝海/红海/衰退期） |
| 情绪共鸣度 | 8% | 是否切中当下社会集体情绪（焦虑、逆袭、复仇、甜宠） |
| 平台适配性 | 5% | 是否符合红果/抖音/快手等平台当前算法偏好与内容政策 |

### 内容层（权重 50%）★ 创作 agent 的主战场
| 指标 | 权重 | 评估标准 |
|---|---|---|
| 钩子强度（前3集） | 14% | 第1集是否有「反常识冲突」或「强悬念」；第3集是否完成首次情绪高潮 |
| 爽点密度 | 12% | 每集（1-2分钟）是否包含至少1个情绪爆点；付费卡点是否落在情绪峰值 |
| 人物弧光 | 10% | 主角是否有清晰成长线；反派是否有足够压迫感；配角是否工具人化 |
| 剧情节奏 | 10% | 是否遵循「压抑-反转-再压抑-大反转」的短剧节拍；是否存在废戏 |
| 台词记忆点 | 4% | 是否有可传播的金句/名场面（利于二创切片引流） |

### 制作层（权重 15%）
| 指标 | 权重 | 评估标准 |
|---|---|---|
| 场景可控性 | 6% | 主场景数量是否 ≤ 5 个；是否存在多场景（战争、大场面） |
| 制作周期 | 4% | 预估制作天数是否 ≤ 14 天 |
| 后期复杂度 | 3% | AI 生成内容成本是否可控；打斗、群戏、宏大场景等数量 |
| 资产复用度 | 2% | 是否可复用公司 AI 资产库内现有资产（古装头套、现代职场套装等） |

### 商业层（权重 10%）
| 指标 | 权重 | 评估标准 |
|---|---|---|
| 付费转化设计 | 5% | 卡点位置是否合理；免费集数与付费集数比例 |
| IP 延展性 | 2% | 是否可开发第二季/系列化 |
| 切片投流适配度 | 2% | 是否有可传播的金句/名场面（利于二创切片引流） |
| 出海适配 | 1% | 剧情是否具备文化普适性；是否易于本地化改编 |

## 计算口径（canonical）
- **汇总只走维度权重**：总分 = Σ(维度得分 × 维度权重)，维度权重 = 市场 25% /
  内容 50% / 制作 15% / 商业 10%。
- **指标权重是维度内相对权**：各指标的百分数只用于在**维度内归一**后加权出维度
  得分（市场层四指标名义和为 30%，维度内归一使用——忠实原 xlsx，原表即如此）。
- **忠实来源注记**：原表标题「四维十六指标」，实际列出 **17 项**指标——本文件忠实
  保留原表名与原权重数字，不作改动。
- **evaluator 只允许维度路径汇总，禁止指标权重直加**（指标百分数直加会因市场层
  名义和 30% ≠ 维度权重 25% 而得出错误总分）。

## 关键判断红线（任一触发 ⇒ 按规定处置，不看总分）

**红线编号以本清单为准**：①受众画像（机内）②题材打压③情绪过时④完播率
（②③④=依赖外部数据的三条）⑤卡点落差⑥主角被动⑦合规。

- 受众画像模糊 → **直接降级**（机内断言：立项 interview 必填画像，add-script 入口预防）
- 题材处于平台打压期（如过度同质化宫斗）→ **一票否决**（依赖 market-watch 带日期评估；
  无数据 ⇒ 升级操作者裁决 human-park，不走修订票）
- 情绪过时（如 2023 年的「赘婿打脸」到 2026 年已审美疲劳）→ **需重构**（同上依赖 market-watch）
- 前 3 集完播率预估 < 45% → **不通过**（机内为**结构代理断言**：第 1 集反常识冲突/
  第 3 集首次高潮/尾钩强度序列；真实完播率投放后回填「待实测」栏）
- 付费卡点情绪落差不足 → **需重写**（机内断言：R4.5 结构；卡点集号从 config.paywall 读取）
- 主角被动超过 30% 戏份 → **改人设或改结构**（机内断言：节拍单主动性字段 →
  story-state 逐集累计 → doctor 滑窗 10 集监控，红线前移为随集报警）
- **合规红线（craft-rules R10a：违法未惩/价值观/敏感题材/平台政策）→ 一票否决 +
  human-park**（行业失败第一梯队，每道门都查——rubric 原表未列，writing-loop 增补为第 7 线）

**数据依赖纪律**：市场层四指标与红线②③④需要外部数据（榜单/政策/投放）。evaluator
打分必须引用 market-watch 的带日期评估（超 2 周 ⇒ inconclusive）；无数据源时输出
「无法评估 + 置疑」，禁止用模型先验编造分数。

## 定级与决策
总分 = Σ(维度得分 × 维度权重)，换算百分制。

| 等级 | 分数区间 | 决策建议 | 资源配置 | 备注 |
|---|---|---|---|---|
| S+ | 90-100 | 立即立项，顶配资源 | 头部内容制作标准，预算上浮 20% | 公司战略级项目 |
| S | 80-89 | 优先立项，快速推进 | 主力团队，标准预算 | 季度核心产出 |
| A | 70-79 | 有条件立项，需优化 | 指定修改项，修改后复评 | 潜力项目，需打磨 |
| B | 60-69 | 储备观察，暂不投入 | 入 IP 库，等待题材窗口期 | 长期储备 |
| C | <60 | 否决 | 归档，不投入资源 | 放弃或重写大纲 |

## 评估报告结构（模板见 templates/evaluation-report.md）
一、项目基本信息（剧名/题材/集数/一句话梗概/核心卖点）
二~五、四个维度逐指标打分 + **评估依据（必须写具体判断，引用集数/场号证据）**
六、综合评分与定级
七、评估结论：核心优势 / 关键风险与修改要求 / 资源配置建议 / 复评节点
