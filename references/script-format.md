# 剧本格式规范（script-format）

> 依据 22 部行业示例剧本的六组格式分析归纳（原始报告：`docs/RESEARCH/fmt-*.md`）。
> 本文件是 episode-writer 自检门与 reviewer 审读门的**机械校验依据**：凡标注「硬」的
> 规则可机器判定，违反即 fail；标注「软」的规则由审读判断。

## 1. 交付物结构 — 「规格书 + 正文」

一部剧的完整交付物 = **剧本头（Bible 摘要）+ 正文**。行业的「一卡剧本」交付物 =
剧本头 + 第一付费卡点前的全部正文（通常前 10 集）——writing-loop 的第一个里程碑
「一卡包」即此物，用于先投放测数据、达标再续产。

剧本头字段（由 `bible/` 与 `outline.md` 汇编生成，不手写第二份）：

| 字段 | 要求 | 示例来源 |
|---|---|---|
| 片名 + 备选片名 | 主片名 1 + 备选 2-3（供 A/B 测试） | 《神兽公园》3 备名 |
| 对标剧 + 热度 | 已验证的同类爆款 + 热度值 | 《夜航船》对标 4188w |
| 题材标签 | 3-5 个（如 豪门复仇/双向救赎/悬疑爱情） | 一卡剧本通例 |
| 类型 / 单集时长 / 总集数 / 免费集数 | 数值 | 2min · 60集 · 免费10集 |
| 一句话梗概 | ≤60 字，含人设反差与钩子 | 「被继母流放十八年…最趁手的一把刀」 |
| 故事简介 | 300-500 字，**允许剧透到底**（给平台/投资方看的） | 通例 |
| 核心策划点 | 核心情绪 / 期待感 / 爽点机制 三段 | 男频长篇通例 |
| 世界观 / 力量体系 | 如适用；含战力表与「表现规则」（见 §5） | 《焚天剑神》 |
| 人设卡 | 见 §2 | 通例 |
| 分段大纲 | 按集区间的宏观走向（1-8 / 8-23 / …） | 《村霸》迷你圣经 |

## 2. 人设卡规格（`bible/characters.md`）

每个具名角色一张卡，字段：

- **基础**：姓名 / 性别 / 年龄 / 职业 / 性格 / **功能**（推动情节・衬托主角・给信息——
  写不出功能的角色应删除或合并）。
- **视觉 token**（硬）：发型、服装、体貌到可直接喂图像模型的粒度（「碎发遮眼，常穿
  连帽卫衣」「cm 级身高、明星参考」）。多形态角色写**形态状态机**：
  `【咸鱼态】…【合体态】…【杀神态】…`，并标注视觉断点集号（「觉醒 80%（32集）」）。
- **voice 规则 + 台词范例**（软）：该角色说话的句式、口头禅、禁忌词，附 2-3 句范例
  ——防多集串味的锚点。
- **核心困境**：一句话（「她渴望被爱，却早已不信任何人」）。
- **弧光声明**：起点 → 终点 + 转折集区间。前置声明，验收时对照。
- **表演要点**：给演员/生成模型的表演指令。

数量约束（硬）：核心人物 3-5 人；有名字角色全剧 ≤20（一卡剧本 ≤8）；
群演一律代号化（`游客A`、`秦家人*3`），禁止给一次性角色起名。

## 3. 正文语法（硬——机器可校验的骨架）

```
第X集                          ← 集标记独立一行；可带钩子式标题：第12集（新婚夜怪事连连）
                                  卡点集在标题标注：第9集（一卡）
X-Y 地点 日/夜 内/外            ← 场景头四要素齐全；X=集号 Y=场序
人物：角色甲、角色乙、群演*3    ← 每场强制人物清单（拍摄/生成的资产调度单）
▲ 一行动作 = 一个镜头。         ← 动作行前缀 ▲（或 △/∆，项目内统一）
▲ 【特写】箱盖缝隙渗出暗红色液体。   ← 生产标注内联：【特写】【特效】【音效】【BGM】
                                  【字幕：三年后】【人名条】【系统面板】
角色甲（情绪/动作提示）：台词。  ← 对白；情绪前缀 100% 必带
角色乙（OS）：内心独白。        ← OS=内心；VO=画外音/旁白/电话；【字幕】=视觉文字
【插入闪回】 … 【闪回结束】      ← 闪回成对标签；回放既有场景用场号引用：【回忆闪回26-3】
▲ 【画面定格】反派扭曲的脸。     ← 集尾可用定格；尾钩必须存在（见 craft-rules R1）
```

数值约束（硬）：

| 项 | 真人短剧 profile | AI 漫剧 profile | 出海英文 profile |
|---|---|---|---|
| 单集字数 | 900-1300 | 550-850 | 500-800 词 |
| 单集场数 | 1-4（均值 1.5-3） | 1-3（8-15 个【画面】卡） | 1-3 |
| 台词 | ≤25 字/句，≤3 行/段 | 同左 | ≤2 sentences |
| OS/旁白密度 | 0-2 条/集 | 4-8 条/集（兼容「听剧」，允许旁白复述画面） | 收敛，靠表演 |
| 总集数惯例 | 付费 80-100 / 免费 80+ | 36-60 | 60-80 |
| 卡点 | 卡一 8-12 集，卡二 26-30，卡三 ~60；备卡制 | 卡一恒为第 10 集 | 订阅制，无硬卡点，每分钟一个情感钩 |

AI 漫剧 profile 附加：正文以**【画面】为原子单元**——每个【画面】是一段自足的静态
视觉描述（≈一张生成图卡，6-12 秒），五轨分离：【画面】/【特写】/【音效】/【字幕】/台词。

## 4. 单集机读头部块（writing-loop 新增，硬）

每个 `episodes/ep-NNN.md` 以 YAML frontmatter 开头，是 doctor 机器审计与
生成指纹（可追溯 + 防「已过门工件静默改写」）的载体：

```yaml
---
ep: 12
arc: arc-02
beat-card: arcs/arc-02.md#ep-012      # 本集节拍单指针
beat-card-hash: a3f29c01d4e2           # 写作时刻 arc 文件的内容哈希（sha256 前 12 位）——
                                       #   doctor 比对当前 arc 文件即得「依据已过期」的集清单
hook-type: H1                          # 集尾钩型（craft-rules R1 八类）
words: 1043
foreshadow-ops: [refresh F-03]         # 本集账本操作（plant/refresh/payoff + ID）
keystone: paywall-1-minus-1            # 关键集标记（无则省略）
mode: direct-write                     # 仅升级重写票标注（fail 计数的机械载体）
written-by: episode-writer (run 3f2a)  # 生成指纹
model: sonnet/high                     # 模型/档位指纹——doctor 审计季内混源断层
rules-version: craft-rules@1 script-format@1
---
```

自检门校验（机器，格式门定位——**叙事实质验收是 reviewer 的带引文断言，机读块
自报字段不作叙事证据**）：frontmatter 完整、`hook-type` 与正文尾钩一致、
`foreshadow-ops` 与 `ledgers/foreshadow.md` 的当集条目一致、`words` 在 profile
字数带内、场景/具名角色 ∈ `ledgers/production.md` 注册表。

## 5. 一致性 spec（防跨集漂移）

- **战力表 + 表现规则**（`bible/world.md`）：每个力量层级配可拍摄的表现规则
  （如「同级打斗 ≥10 拍，跨大级秒杀 ≤3 拍」「打斗时长=实力差」）。审读按表校验，
  禁止战力忽高忽低。
- **数字锚点**：贯穿性数值（欠款五十万、心跳 60、融合率 80%）在账本登记，
  每次出现必须与 `ledgers/story-state.md` 当前值一致。
- **重叠帧承接**：下集开场重放上集末 1-2 拍（可逐字重复）后再推进——集间承接的
  默认手法；跳时间用【字幕：三天后】显式声明。

## 6. 格式反面 lint（审读门快查）

1. 文学化开头（环境描写/抒情铺垫开场——新人第一死因；第 1 场必须是冲突或悬念画面）。
2. 心理描写残留（改编常见病：小说式内心活动未译成 OS/表演/对手戏）。
3. 一行多镜头（动作行含两个以上主语-动作对，应拆行）。
4. 无情绪前缀的对白；超长台词（>3 行）。
5. 同一事实的议论 VO >2 轮（craft-rules R6.2）。
6. 场景头缺要素；人物清单与实际出场不符。
7. 具名角色超编；新场景未在细纲预算内（制作层红线：主场景 ≤5）。
8. 卡点集无标注；尾钩缺失或与 frontmatter 声明钩型不符。
